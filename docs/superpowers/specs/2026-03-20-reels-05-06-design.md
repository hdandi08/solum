# SOLUM Reels 05 & 06 — Design Spec

> **Series:** SOLUM Instagram Animated Reels — Education Arc
> **Format:** Self-contained HTML files, 405×720px (9:16), ~21 seconds
> **Reference implementation:** `artefacts/solum-reel-04-animated.html`
> **Production standards:** `memory/feedback_reel_production_standards.md`
> **Legibility priority:** All text must be large, high-contrast, and readable on a phone screen. Zero decoratively small text.

## Reel 05 — The Scalp

**File:** `artefacts/solum-reel-05-animated.html`

### Messaging

- **Wrong:** You shampoo your scalp. You don't clean it.
- **Why:** Shampoo runs over it, you rinse. That's wetting, not working.
- **Fix (implied):** 2 minutes of proper massage is all it takes.
- **Proof:** What builds up on an unmassaged scalp — sebum, dead skin, product residue, blocked follicles, bacteria.

### Scene Breakdown

```

YOUR SCALP IS SKIN.
YOU'VE NEVER ONCE CLEANED IT.
Font: 72px Bebas Neue (.t-hook class, no override needed)
```

### Animation Notes — Reel 05

- **S3 scroll:** Identical to Reel 04 ingredient list. `padding-top: 700px` on list. CSS vignette mask on wrap. Class `.scroll-running` added by JS on scene activation, removed in `restartReel()`. `revealTimer` stored separately, cancelled in `restartReel()`. `reveal-active` removed from `#s3` on both `showScene()` transition and `restartReel()`.
- **S4 stagger:** Single `.txt.pos-centre` wrapper. `.s4-line2` child with `opacity: 0; transition: opacity 0.5s, transform 0.5s; transition-delay: 0.5s`. Trigger: `#s4 .txt.visible .s4-line2 { opacity: 1; transform: translateY(0); }`.
- **Highlight item:** BACTERIA — use `<span id="bacteria">` inside the scroll list. Triggered by `reveal-active` class on `#s3`:
  ```css
  #s3.reveal-active .ingredient-list span.highlight { opacity: 1; color: var(--steel-blue); font-weight: 700; font-size: 22px; transform: scale(1.06); }
  #s3.reveal-active .ingredient-list span#bacteria { transition-delay: 0.4s; }
  ```
  Do NOT use a second setTimeout for this — CSS transition-delay handles the stagger.

### Animation Notes — Reel 06

- **Countdown implementation:**
  ```js
  let countdownTimers = []; // ARRAY — stores all setTimeout IDs so all can be cancelled on restart
  // In showScene(3):
  countdownTimers = [];
  const nums = ['3','2','1','0'];
  const delays = [0, 1000, 2000, 3200];
  const counterEl = document.getElementById('countdown-num');
  nums.forEach((n, i) => {
    countdownTimers.push(setTimeout(() => {
      counterEl.textContent = n;
      if (n === '0') {
        counterEl.classList.add('timer-pulse');
        countdownTimers.push(setTimeout(() => {
          counterEl.classList.add('timer-done'); // fades timer out (opacity: 0, transition 0.4s)
          document.getElementById('s3-stat').classList.add('stat-visible'); // fades stat in
        }, 400));
      }
    }, delays[i]));
  });
  ```
- **Reset in `restartReel()`:**
  - `countdownTimers.forEach(id => clearTimeout(id)); countdownTimers = [];`
  - Reset `#countdown-num` textContent back to `'3'`
  - Remove `.timer-done` from counter element (restores opacity)
  - Remove `.stat-visible` from `#s3-stat` (hides stat block again)
  - Remove `.timer-pulse` from counter element
- **S4 stagger:** Same as Reel 05 — `.s4-line2` child, `id="s4"`, CSS trigger `#s4 .txt.visible .s4-line2`.
- **70% number:** Use `color: var(--steel-blue)` on the "70% MORE MOISTURE" line to make the stat pop. This is the legibility anchor for the scene.

Every scene must include:
- `<div class="brand-rule"></div>` — 3px steel blue left edge

- `.txt.pos-centre` wrapper with `data-delay` for all readable text content

### JS State Cleanup (both reels)

`restartReel()` must cancel ALL timers and reset ALL animated state. For Reel 05 this means cancelling `revealTimer` and removing `scroll-running` + `reveal-active`. For Reel 06 this means calling `countdownTimers.forEach(id => clearTimeout(id))` and resetting the countdown display + stat visibility. Both reels follow the same base engine as Reel 04.
